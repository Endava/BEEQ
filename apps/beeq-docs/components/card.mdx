Cards group related content and actions into a single scannable surface. Use them to present highlights, supporting details, or compact summaries without forcing a rigid internal layout.

<Note>
  `bq-card` does not provide a default content layout. The project integrating the component is responsible for arranging the content inside it.
</Note>

## When to use

<CardGroup cols={2}>
  <Card>
    <span className="flex items-center mr-2 text-lg font-medium" role="heading">
      <Icon className="mr-2" icon="thumbs-up" iconType="solid" size={20} color="var(--bq-stroke--success)" />
      Use cards when
    </span>

    * You want to present concise information, actions, or collections of related content in a visually organized and scannable format
    * You need to highlight featured details, related content, or navigation options in a contained surface
    * The design benefits from a structured layout that helps users scan and compare grouped content
  </Card>

  <Card>
    <span className="flex items-center mr-2 text-lg font-medium" role="heading">
      <Icon className="mr-2" icon="thumbs-down" iconType="solid" size={20} color="var(--bq-stroke--danger)" />
      Do not use cards when
    </span>

    * You are presenting a single piece of information that does not need additional structure or supporting elements
    * The content does not benefit from a structured card layout
    * The design calls for a more fluid and continuous display instead of separated content blocks
  </Card>
</CardGroup>

## Card patterns

The Card pattern can support multiple presentation styles depending on the content you need to surface.

<CardGroup cols={3}>
  <Card title="Mini card">
    Mini cards are employed when there is a need for a condensed presentation of information, such as summaries or quick references.
    These smaller card variants are useful in scenarios where space is limited, and designers want to maintain a compact and visually appealing layout while still conveying essential details.
  </Card>

  <Card title="Card list">
    Card lists are utilized to organize and display a series of related cards in a structured manner, allowing users to browse through a collection of content or items.
    This design pattern is beneficial when presenting information that follows a consistent format, making it easy for users to scan and navigate through the list.
  </Card>

  <Card title="Card with title and description">
    The card title and description are essential components within a card, serving to provide users with key information about the content or item featured in the card.
    The title succinctly conveys the main topic, while the description offers additional context or details.
  </Card>
</CardGroup>

## Anatomy

<Frame className="px-4 py-4" caption="Card anatomy">
  <img src="https://placeholdit.com/600x400" alt="Placeholder for the Card anatomy image" />
</Frame>

Cards are flexible containers, so their anatomy can vary depending on the content. In most cases, a card includes a container, an optional visual element, a content area, and optional supporting actions.

| Part | Element | Description |
|---|---|---|
| **1** | Container | The outer surface that groups the content into a single, scannable unit |
| **2** | Visual | An optional icon, image, or highlight area that helps users identify the card content quickly |
| **3** | Content area | The main body containing the title, description, metrics, or supporting text |
| **4** | Actions | Optional buttons or links used when the card includes follow-up actions |

## Component options

### Types

`type` defines the card padding behavior:

| Type | When to use |
|---|---|
| **default** | Use for most cards when the component should provide its own internal padding |
| **minimal** | Use when the content inside the card needs full control over spacing, such as mini cards or media-led layouts |

<Tip>
  `default` is the standard option and applies the component's built-in padding automatically.
</Tip>

### Border radius

Use the `border` property to control the card corner radius.

## Accessibility

- Provide descriptive alt text for images within cards.
- Implement proper heading structures so card content remains easy to scan and understand.
- Ensure that interactive elements within cards are navigable using keyboard controls.
- Maintain sufficient color contrast between the card background, text, and supporting UI elements.
- Regularly test card implementations with accessibility tools to identify and address potential issues.

## API reference

### Properties

| Property | Attribute | Description | Type | Default |
|---|---|---|---|---|
| `border` | `border` | The corner radius of the card component | `'none'` \| `'xs2'` \| `'xs'` \| `'s'` \| `'m'` \| `'l'` \| `'full'` | `'m'` |
| `type` | `type` | Type of card component | `'default'` \| `'minimal'` | `'default'` |

### Slots

| Slot | Description |
|---|---|
| *(default)* | The content of the card component |

### Shadow parts

| Part | Description |
|---|---|
| `wrapper` | The wrapper container `<div>` of the element inside the shadow DOM |
