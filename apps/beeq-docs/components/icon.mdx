Icons provide visual cues that help users quickly understand actions, statuses, navigation, and meaning across an interface. They play a crucial role in enhancing the user experience by offering concise metaphorical communication while maintaining a cohesive design language throughout the system.

<Note>
  BEEQ icons are based on the [Phosphor icons library](https://phosphoricons.com/), a flexible open-source icon family licensed under MIT. Before using named icons in an app, make sure your SVG assets are served from a public path and configured with `setBasePath()`.
</Note>

## When to use

  <Card>

      <Icon className="mr-2" icon="thumbs-up" iconType="solid" size={20} color="var(--bq-stroke--success)" />
      Use icons when
    </span>
    - You need clear and concise visual communication
    - An action, status, or concept benefits from a familiar visual cue
    - The icon emphasizes key information without replacing necessary text
    - Consistent iconography improves scanning and recognition across the UI
  </Card>
  <Card>
    <span className="flex items-center mr-2 text-lg font-medium" role="heading">
      <Icon className="mr-2" icon="thumbs-down" iconType="solid" size={20} color="var(--bq-stroke--danger)" />
      Do not use icons when
    </span>
    - They add visual noise without improving clarity
    - The meaning is too ambiguous to stand on its own
    - Several icon styles are mixed in the same interface
    - Color or icon shape would become the only way to understand important information
  </Card>
</CardGroup>

## Icon types

The design system supports multiple icon styles, but the most common visual distinction is between outlined and filled icons.

<CardGroup cols={2}>
  <Card title="Outlined">
    Outline icons serve as minimalist visual cues, providing a sleek and clean appearance in digital interfaces. Their usage follows a consistent design language and helps reduce visual noise in general-purpose UI.
  </Card>
  <Card title="Filled">
    Filled icons are useful for specific actions or elements that require more emphasis or interaction. Use them thoughtfully for moments like favorites, ratings, or selected states where stronger visual weight helps.
  </Card>
</CardGroup>

`bq-icon` renders a small shadow-DOM wrapper and injects a sanitized SVG into it. The component itself stays intentionally simple so it can be reused inside buttons, alerts, inputs, menus, and other higher-level components.

| Part | Element | Description |
|---|---|---|
| **1** | Base | The component's internal wrapper that holds the icon SVG content |
| **2** | SVG | The rendered `<svg>` element inside the component |

## Common usage patterns

### Default

Use the default icon when you want a concise and flexible visual element for navigation bars, toolbars, buttons, menus, and inline UI cues.

  The default icon size is `24px`, so you only need to set `size` when you want something smaller or larger.

### Styles

You can choose from multiple icon styles by adding a style suffix to the icon name, such as `-thin`, `-light`, `-bold`, `-fill`, or `-duotone`. The default unsuffixed name is the regular style. This is the preferred approach over the legacy `weight` prop.

Use the `src` prop when you want to load a custom icon from a local or CORS-enabled endpoint. When `src` is set, the `name` prop is ignored.

### Asset base path and custom icon libraries

Named icons are loaded as SVG files. If your app copies BEEQ's icons into a public directory, point the component to that URL with `setBasePath()`.

<CodeGroup>
  ```ts TypeScript icon="typescript"
  import { setBasePath } from "@beeq/core";

  setBasePath("/icons/svg");
  ```

  ```tsx React icon="react"
  import React from "react";
  import ReactDOM from "react-dom/client";
  import { setBasePath } from "@beeq/core";
  import App from "./App";

  setBasePath("/icons/svg");

  ReactDOM.createRoot(document.getElementById("root")!).render(
    <React.StrictMode>
      <App />
    </React.StrictMode>,
  );
  ```
</CodeGroup>

If you want to use an alternative icon library, point `setBasePath()` to a local or remote SVG directory that follows the file naming convention you plan to use.

```ts
import { setBasePath } from "@beeq/core";

setBasePath("https://cdn.jsdelivr.net/npm/heroicons@2.1.5/24/outline");
```

<Tip>
  Make sure the icon names used in `name` match the files available at the configured path. If you only copy a subset of icons, keep the original file names unchanged.
</Tip>

## Accessibility

- **Label meaningful standalone icons** — use the `label` prop when the icon itself conveys meaning and needs to be announced to assistive technologies.
- **Prefer text with icons for ambiguous actions** — some icons do not make sense by themselves, so add a visible label whenever clarity matters.
- **Use proper interactive wrappers** — if an icon is clickable, give it a generous hit area by placing it inside [Button](/components/button) instead of relying on the icon alone.
- **Do not rely on color only** — status and meaning should still be understandable without perceiving color differences.
- **Keep naming consistent** — use familiar metaphors and stable icon names so designers, developers, and users build the same expectations.

## API reference

### Properties

| Property | Attribute | Description | Type | Default |
|---|---|---|---|---|
| `color` | `color` | Set the stroke color of the SVG. The value should be a valid palette token | `string` | `undefined` |
| `label` | `label` | Label for the icon, used for accessibility | `string` | `undefined` |
| `name` | `name` | Icon name to load | `string` | `undefined` |
| `size` | `size` | Set the size of the SVG | `number` \| `string` | `24` |
| `src` | `src` | Set the source of the SVG. If set, `name` is ignored | `string` | `undefined` |
| `weight` | `weight` | **Deprecated.** Legacy icon weight/style prop | `'thin'` \| `'light'` \| `'regular'` \| `'bold'` \| `'fill'` \| `'duotone'` | `undefined` |

### Events

| Event | Description | Type |
|---|---|---|
| `svgLoaded` | Fires when the SVG content has loaded | `CustomEvent<any>` |


  ```vue Vue icon="vuejs"
  <BqIcon @svgLoaded="onSvgLoaded($event)" />
  ```
</CodeGroup>
