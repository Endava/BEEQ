An option is the individual selectable item inside an option list. It provides the row-level label, prefix and suffix slots, selection state, and option events used by higher-level patterns such as dropdowns, grouped lists, and select-like menus.

<Note>
  `bq-option` is a building block, not a standalone menu pattern. Use it with `bq-option-list`, and pair it with [Dropdown](/components/dropdown) when you need a trigger and floating panel. For grouped choices, combine it with `bq-option-group`.
</Note>

## When to use

      Do not use `bq-option` when
    </span>
    - You need the whole trigger-and-panel behavior of a [Dropdown](/components/dropdown)
    - You need a complete form control pattern such as [Select](/components/select)
    - The UI would be clearer as radios, tabs, visible buttons, or another always-visible control
    - You expect `bq-option` to manage list behavior on its own without `bq-option-list` or a parent pattern
  </Card>

## Composition patterns

`bq-option` stays intentionally small. The surrounding container decides how options are presented, grouped, opened, and persisted.

An option is an interactive item rendered as a button inside an option list. It can include optional prefix or suffix content, while the label remains the primary piece of information users scan and select.

| Part | Element | Description |
|---|---|---|
| **1** | Base | The interactive button surface that receives focus and selection |
| **2** | Prefix | Optional leading content, usually an icon |
| **3** | Label | The main option text shown in the default slot |
| **4** | Suffix | Optional trailing content, such as an icon or status cue |

## Accessibility

- **Keyboard access** â€” users should be able to reach the dropdown trigger with <kbd>Tab</kbd>, open the list with <kbd>Enter</kbd> or <kbd>Space</kbd>, and move through options with <kbd>Up</kbd> and <kbd>Down</kbd>.
- **Selection behavior** â€” a focused option can be selected with <kbd>Enter</kbd> or <kbd>Space</kbd>. In many dropdown flows this may also close the panel unless the container is configured to stay open.
- **Close behavior** â€” dropdown-based option lists should close predictably with <kbd>Esc</kbd> or when focus leaves the interaction.
- **Multi-select guidance** â€” in multi-select dropdown patterns, deselection should remain keyboard accessible and consistent with the host component behavior.
- **Readable labels** â€” each option label should clearly communicate the result of selection without depending on icon-only meaning.

## API reference

### `bq-option` properties

| Property | Attribute | Description | Type | Default |
|---|---|---|---|---|
| `disabled` | `disabled` | If `true`, the option is disabled | `boolean` | `false` |
| `displayValue` | `display-value` | The display value of the option. It can be used to override the default displayed value | `string` | `undefined` |
| `hidden` | `hidden` | If `true`, the option is hidden | `boolean` | `false` |
| `selected` | `selected` | If `true`, the option is selected and active | `boolean` | `false` |
| `value` | `value` | A string representing the value of the option | `string` | `undefined` |

### `bq-option` events

| Event | Description | Type |
|---|---|---|
| `bqBlur` | Fires when the item loses focus | `CustomEvent<HTMLBqOptionElement>` |
| `bqClick` | Fires when the item is clicked | `CustomEvent<HTMLBqOptionElement>` |
| `bqEnter` | Fires on Enter key press | `CustomEvent<HTMLBqOptionElement>` |
| `bqFocus` | Fires when the item receives focus | `CustomEvent<HTMLBqOptionElement>` |

### `bq-option-list` properties

| Property | Attribute | Description | Type | Default |
|---|---|---|---|---|
| `ariaLabel` | `aria-label` | Aria label for the list | `string` | `'Options'` |

### `bq-option-list` events

| Event | Description | Type |
|---|---|---|
| `bqSelect` | Fires when a `bq-option` is selected by click or Enter | `CustomEvent<{ value: string; item: HTMLBqOptionElement }>` |

<Note>
  `bq-option` emits low-level interaction events (`bqClick`, `bqEnter`, `bqFocus`, `bqBlur`), while `bq-option-list` emits the higher-level `bqSelect` event most applications will use.
</Note>

  ```html Angular icon="angular"
  <bq-option-list (bqSelect)="onSelect($event)"></bq-option-list>
  ```

  ```vue Vue icon="vuejs"
  <BqOptionList @bqSelect="onSelect" />
  ```
</CodeGroup>
