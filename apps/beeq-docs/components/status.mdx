The Status component is a UI element that communicates the status of an item, task, or process to the user. Statuses are commonly used to indicate the current state of an item, such as completed or pending, or to provide updates on the progress of a task or process.

<Note>
  The original design guidance covers both dot status and icon status patterns. The current `bq-status` component implements the dot-and-label variant: a colored circular indicator paired with descriptive text.
</Note>

## When to use

<CardGroup cols={2}>
  <Card>
    <span className="flex items-center mr-2 text-lg font-medium" role="heading">
      <Icon className="mr-2" icon="thumbs-up" iconType="solid" size={20} color="var(--bq-stroke--success)" />
      Use statuses when
    </span>
    - You need to indicate the state or condition of an element or action
    - Users benefit from quick visual understanding of progress, availability, or outcome
    - The UI should provide updates about changes in a task, item, or process
    - A short label plus a visible indicator improves scanning in lists, tables, and detail views
  </Card>
  <Card>
    <span className="flex items-center mr-2 text-lg font-medium" role="heading">
      <Icon className="mr-2" icon="thumbs-down" iconType="solid" size={20} color="var(--bq-stroke--danger)" />
      Do not use statuses when
    </span>
    - The information is already clear without an additional indicator
    - Status changes are minimal or rare and the extra UI adds noise
    - Screen space is too limited for both a clear label and indicator
    - The state is constant and does not help the user make decisions
  </Card>
</CardGroup>

## Status types

Status components are useful for providing users with a quick and visual understanding of the situation or status of an element within the system.

The shipped component keeps the structure intentionally simple: a colored status dot and a text label. This makes it easy to scan in tables, lists, cards, and other dense layouts.

| Part | Element | Description |
|---|---|---|
| **1** | Base | The internal wrapper for the status component |
| **2** | Circle | The colored circular indicator that marks the status type |
| **3** | Text | The text label that explains the current status |

## Variants

Use `type="alert"` when you want to draw immediate attention to a critical or urgent message. This status provides a noticeable visual cue, ensuring users quickly recognize the importance of the displayed information.

## Accessibility

- **Always pair the indicator with text** — statuses should be accompanied by descriptive text so meaning does not depend on color alone.
- **Choose colors with sufficient contrast** — the label must remain readable and the visual indicator should be distinguishable in context.
- **Use consistent language** — clear, stable labels reduce confusion and help assistive technology users understand changes more easily.
- **Provide both visual and textual feedback** — when a status changes, users should be able to understand the update whether they rely on visuals, text, or both.
- **Test the component in context** — validate that the status remains understandable inside tables, cards, forms, and other composite layouts.
