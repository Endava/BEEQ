The grid provides the fundamental groundwork for placing visual elements. Working on the grid ensures a standard direction for creative decision-making across products and a responsive framework for design on different devices. It makes an app look coherent, enhances visual hierarchy, and balances design.

There is no one-size-fits-all grid behavior. Different content types can render better when using fixed, stretch, or hybrid grid models. Use the following guidance to decide on the best approach for your application.

---

## Grid anatomy

All grids are made up of three elements: columns, gutters, and margins.

### Columns

Columns are the building blocks of a grid and mark where elements should be placed. A 12-column framework is common for its flexibility and easy division. It is highly composite for layouts and can be updated responsively based on screen size.

### Gutters

Gutters are the negative space between columns. Their width should be a multiple of the base unit. To better adapt to a given screen size, gutter widths can change at different breakpoints.

### Margins

Margins are the space outside of the grid columns and rows. Margin widths can be fixed or percentage-based and can also change at different breakpoints.

## Grid types

There are many ways to combine columns, gutters, and margins to create different grid layouts. Consistency is key to building familiar patterns that make your app easy to scan and navigate. A good grid will also adapt to different screen sizes and orientations, ensuring cohesion across environments.

Aligning baselines to a specific absolute grid establishes a vertical rhythm — a pattern that is easier for the human brain to scan. Baseline grids are especially useful with content that spans multiple columns.

## Breakpoints

The BEEQ Design System uses the following breakpoints to optimize layouts across screen sizes. Not all breakpoints need to be used in a product. Breakpoints only need to be referenced when the design needs to change for a particular breakpoint range.

| Breakpoint     | Viewport        |
| -------------- | --------------- |
| Phones         | 576 – 768 px    |
| Tablets        | 768 – 992 px    |
| Desktop        | 992 – 1200 px   |
| Large devices  | 1200 – 1920 px  |

## Layout

Layouts are a culmination of defined rules and intentional organization of content. Bringing your content into thoughtful structures is key, but making it all flow together with a clear hierarchy across platforms and screen sizes requires scaling logic. Individually, adaptive and responsive design can each address this challenge. In some cases, a mix of both is the right choice.

### Responsive

Responsive design uses a single layout where content is fluid and can adapt to changing format sizes. This technique uses media queries to inspect a given device's characteristics and renders content accordingly. Responsive design allows you to build a feature once and expect it to work across all screen sizes.

### Adaptive

An adaptive layout changes entirely based on the format it is presented in. Adaptive design has multiple fixed layout sizes and triggers the browser to load a given layout based on the available space. Sites built with adaptive design use media queries to read breakpoints — similar to responsive design — plus additional markup based on the device's capabilities. This process is known as "progressive enhancement."

## Container type

Understanding the distinctions between fluid and fixed layouts empowers designers to make informed decisions based on project requirements. In practice, the intersection of fluid and fixed layouts is often used — seeking a balance between adaptability and precision. Striking this balance ensures visually captivating designs that function seamlessly across an array of devices.

<CardGroup cols={2}>
  <Card title="Fluid">
    Fluid layouts are calculated to their relative size and stretch as the screen (viewport) is resized. For example, if you set the layout width to 100%, each column will only be calculated to its relative size and will stretch as the browser is resized. Fluid layouts are often used to create a seamless experience across various devices.

## Usage guidelines

<CardGroup cols={2}>
  <Card>
    <span className="flex items-center mr-2 text-lg font-medium" role="heading">
      <Icon className="mr-2" icon="thumbs-up" iconType="solid" size={20} color="var(--bq-stroke--success)" />
      Do
    </span>

    - Keep layout regions within column boundaries — never extend them into the gutters
    - Use gutters to create breathing room between your layout regions
    - Apply breakpoints only when the design needs to change for a specific viewport range
    - Prefer semantic tokens (`--bq-spacing-*`) for margin and gutter values to stay aligned with the design system

  </Card>
  <Card>
    <span className="flex items-center mr-2 text-lg font-medium" role="heading">
      <Icon className="mr-2" icon="thumbs-down" iconType="solid" size={20} color="var(--bq-stroke--danger)" />
      Don't
    </span>

    - Don't place content inside the gutters — they exist exclusively to separate columns
    - Don't mix different grid types within the same screen without a clear visual separation
    - Don't use arbitrary pixel values for spacing — rely on the base unit multiples defined by the system
    - Don't define breakpoints for every possible screen size — focus on the ranges that meaningfully change the layout

  </Card>
</CardGroup>
