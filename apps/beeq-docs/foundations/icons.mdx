BEEQ Design System uses the [**Phosphor Icons**](https://phosphoricons.com/) set, a comprehensive and versatile collection of icons that cater to various user interface needs. These icons are designed with consistent visual language and style, ensuring a cohesive and aesthetically pleasing user experience.

Icons are essential elements within UI/UX design systems, fulfilling various roles and objectives:

- **Navigation:** Icons act as navigational elements, helping users orient themselves and find their way around an interface.
- **Actions:** Icons visually represent specific actions or operations, such as saving, deleting, printing, or sharing.
- **Status Indicators:** Icons are used to convey the status of a system, process, or object. For instance, a checkmark icon may indicate a successful action or completion.
- **Visual Hierarchy:** Icons contribute to establishing a visual hierarchy within an interface.
- **Brand Identity:** Icons can be designed to reflect a brand's identity and establish consistency across various touchpoints.

## Characteristics

Icons within design systems require careful consideration of overlays, color schemes, sizing, grid alignment, and development practices to ensure consistency, accessibility, and usability across different platforms and contexts.

### Size

Icon size will vary based on the use case and platform. Smaller icons can give information or reinforce an idea, but should not be used for interaction. When choosing size, consider the way someone might move through an experience.

### Structure and grid

Icons in the library use a 24px × 24px frame. The space inside the outer frame is known as the **live area**. The complete composition, including clear space, is known as the **trim area**.

## Usage guidelines

<CardGroup cols={2}>
  <Card title="Streamlining design scannability">
    Icons serve to quickly convey information. Emphasizing clarity ensures immediate understanding, transcending language barriers. Common actions like edit, delete, and search have established icons that are universally understood.

    <Frame caption="Streamlining design scannability">
      <img
        src="/foundations/images/icons/icons-usage-scannability.png"
        alt="Streamlining design scannability"
      />
    </Frame>
  </Card>
  <Card title="Icons should act as complementary visual cues">
    A good way to think about using icons in your design is as complementary visual cues for text. By combining text and icons, you get the best of both worlds. As a general rule, try to include both if you have the space. This is especially important when the action is less common and not a "universal" icon or action that is likely to already be familiar to the user.

    <Frame caption="Icons as complementary visual cues">
      <img
        src="/foundations/images/icons/icons-usage-complementary-cues.png"
        alt="Icons as complementary visual cues"
      />
    </Frame>
  </Card>
  <Card title="Don't scale up icons">
    System icons are made to a specific standard. Changing the size of an icon will greatly diminish its quality. Canvas does not recommend scaling icons. If for a specific reason, you do need to scale an icon we recommend following Canvas spacing rules and resizing in increments of 4 pixels, never going below 20px.

    <Frame caption="Don't scale up icons">
      <img
        src="/foundations/images/icons/icons-usage-dont-scale.png"
        alt="Don't scale up icons"
      />
    </Frame>
  </Card>
  <Card title="Using Tooltips">
    Tooltips are used to provide additional information or context. They should never contain information that is vital to completing a task, as this is hidden from plain sight when tooltips are used. Tooltips can also be used to display an icon's action. It is important to be aware that tooltips are not easily accessible on mobile devices, so labels are preferred where possible.

    <Frame caption="Using tooltips with icons">
      <img
        src="/foundations/images/icons/icons-usage-tooltips.png"
        alt="Using tooltips with icons"
      />
    </Frame>
  </Card>
  <Card title="Icons alignment">
    System icons should be center-aligned vertically or horizontally depending on how they are stacked. This ensures that icons look visually balanced when in close proximity to another icon. The same rule should be followed when text accompanies an icon.

    <Frame caption="Icons alignment">
      <img
        src="/foundations/images/icons/icons-usage-alignment.png"
        alt="Icons alignment"
      />
    </Frame>
  </Card>
  <Card title="Icons SEO">
    Icons distill essential characteristics for quick metaphorical communication. For instance, icons like "home," "house," and "main page" might share the same visual representation, allowing for versatile interpretation.

    <Frame caption="Icons SEO">
      <img
        src="/foundations/images/icons/icons-usage-seo.png"
        alt="Icons SEO"
      />
    </Frame>
  </Card>
  <Card title="Don't overuse icons">
    When using icons it's crucial to strike a balance and avoid overloading the visual with unnecessary symbols. Redundant icons can create visual clutter, making it challenging for users to focus on essential elements. Each icon should serve a distinct and meaningful purpose, contributing to the overall comprehension of the content or functionality. Strive for a clean and streamlined design to facilitate smoother user interactions and enhance overall usability.
