Spacing in BEEQ is built on a 4px grid system. It creates a consistent foundation for layout, positioning, and rhythm across products, helping interfaces feel balanced, organized, and easier to use.

The spacing scale can be applied to padding, margin, and gap values. It supports both internal spacing within components and external spacing between elements, so layouts stay consistent as they scale across screens and use cases.

---

## 4px grid system

In BEEQ, we implement a 4px grid system for spacing to address layout and positioning requirements with consistency. This system improves alignment, scalability, and adaptability across devices and screen resolutions.

The spacing system is categorized using T-shirt sizes (`XS`, `S`, `M`, `L`, `XL`, `XXL`), with each size mapped to a specific token value. These values are exposed through tokens in the BEEQ design system and can be applied consistently across components and layouts.

Spacing serves a dual purpose within components: it is used both inside components (`padding`) and between components (`margin` and `gap`). Internal spacing usually uses smaller values to keep related elements visually connected, while external spacing uses larger values to create separation and improve scanability.

The spacing inside coded components is already defined, allowing you to focus on building layouts for your users. When creating custom components, use the same spacing scale to maintain consistency and a natural flow across pages.

## Spacing scale

Used for spacing **inside** components and elements.

| Token | Value |
| --- | --- |
| `spacing-xs3` | `2px` |
| `spacing-xs2` | `4px` |
| `spacing-xs` | `8px` |
| `spacing-s` | `12px` |
| `spacing-m` | `16px` |
| `spacing-l` | `24px` |
| `spacing-xl` | `32px` |
| `spacing-xxl` | `40px` |
| `spacing-xxl2` | `56px` |
| `spacing-xxl3` | `64px` |
| `spacing-xxl4` | `96px` |
