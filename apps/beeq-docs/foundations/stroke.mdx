By using strokes in a design system, we have the ability to define and accentuate visual elements. The importance of strokes lies in guiding the user. Additionally, stroke properties such as thickness, color, and style play a crucial role in establishing a unified visual language within the design system. Thoughtful use of strokes ensures consistency, reinforces brand identity, and contributes to a polished and professional design.

## Usage guidelines

In our design system, we will utilize:

- Size `s` (1px) for default states such as input.
- Size `m` (2px) for components in focused and active states.
- Size `l` (3px) for larger elements throughout the design system.

All these configurations can also be found in tokens as predefined settings.

## Accessibility

In the context of accessibility, the focus state stroke is a crucial design element aimed at enhancing usability for individuals with varying abilities. When a user interacts with a component, such as a button or a form field, the focus state stroke provides a visible indicator, aiding those using assistive technologies or navigating through the interface using a keyboard.

Choosing an appropriate color for the focus state stroke is paramount for accessibility. It's recommended to use colors with sufficient contrast against the background to ensure visibility for users with low vision or color blindness. Common practices include using vibrant colors that stand out or adhering to established color contrast ratios outlined in accessibility guidelines. This thoughtful approach to color selection contributes to a more accessible and user-friendly design.
