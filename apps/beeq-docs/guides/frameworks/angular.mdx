BEEQ works best in Angular through `@beeq/angular`, the official Angular wrapper around BEEQ web components.

For new Angular applications, we recommend the standalone setup with direct component imports from `@beeq/angular/standalone`. Keep the NgModule setup only for existing module-based apps.

It lets you use BEEQ components directly in Angular templates while keeping familiar Angular patterns for outputs, forms, and component-level imports.

<Columns cols={3}>
  <Card title="Angular-friendly outputs">
    Listen to BEEQ events with Angular event bindings such as `(bqClick)` and `(bqChange)`.
  </Card>
  <Card title="Forms support">
    Use supported BEEQ form controls with `[(ngModel)]` or reactive forms instead of wiring values by hand.
  </Card>
  <Card title="Standalone-first setup">
    Import only the BEEQ components you need in standalone Angular components, with NgModule support still available for older apps.
  </Card>
</Columns>

<Check>
  In the standard Angular integration flow, you do not need to call `defineCustomElements()` yourself. The Angular wrapper handles component registration for you.
</Check>

## Get started

You can add BEEQ to an existing Angular project in a few steps:

<Steps>
  <Step title="Check your project setup" titleSize="h3">
    Make sure your project already has:

    - Angular `14+`
    - Node.js `18+`
    - a working Angular application created with the Angular CLI or an equivalent setup

    If your app is already running, you can usually add BEEQ without changing your overall architecture.
  </Step>

  <Step title="Install the packages" titleSize="h3">
    Install both the core package and the Angular wrapper:

    ```bash
    npm install @beeq/core @beeq/angular
    ```

    What each package gives you:

    - `@beeq/core` includes the web components, styles, icons, and shared types
    - `@beeq/angular` provides Angular bindings, outputs, and value accessors
  </Step>

  <Step title="Add the global styles" titleSize="h3">
    Load the BEEQ stylesheet once for your whole application.

    <Tabs>
      <Tab title="angular.json">
        Add the stylesheet to the Angular build config:

        ```json angular.json
        {
          "projects": {
            "my-app": {
              "architect": {
                "build": {
                  "options": {
                    "styles": [
                      "src/styles.scss",
                      "node_modules/@beeq/core/dist/beeq/beeq.css"
                    ]
                  }
                }
              }
            }
          }
        }
        ```
      </Tab>

      <Tab title="styles.css">
        Or import it in your main stylesheet:

        ```css src/styles.css
        @import "@beeq/core/dist/beeq/beeq.css";
        ```
      </Tab>
    </Tabs>

    <Tip>
      Keep your application stylesheet loaded after the BEEQ stylesheet if you want your own project styles to override shared defaults.
    </Tip>
  </Step>

  <Step title="Configure icons" titleSize="h3">
    Many BEEQ components rely on SVG icons. Your application needs to serve the SVG files and tell BEEQ where to find them.

    <Warning>
      Be aware that **there are over 9k SVG files**, which can slow down your development or build time if you copy the full icon folder into every project.
    </Warning>

    <Tabs>
      <Tab title="angular.json assets">
        Copy the SVG files from `node_modules` into your Angular assets output, then set the public base path during app startup.

        <CodeGroup>
          ```json angular.json highlight={12-19}
          {
            "projects": {
              "my-app": {
                "architect": {
                  "build": {
                    "options": {
                      "assets": [
                        "src/favicon.ico",
                        "src/assets",
                        {
                          "glob": "**/*",
                          "input": "node_modules/@beeq/core/dist/beeq/svg",
                          "output": "assets/svg/"
                        }
                      ]
                    }
                  }
                }
              }
            }
          }
          ```

          ```ts src/main.ts highlight={1,3}
          import { setBasePath } from "@beeq/core";

          setBasePath("/assets/svg");
          ```
        </CodeGroup>
      </Tab>

      <Tab title="CDN or existing icon source">
        If your project already serves icons from another location, you can point BEEQ there instead:

        ```ts
        import { setBasePath } from "@beeq/core";

        setBasePath("https://cdn.jsdelivr.net/npm/heroicons@2.1.5/24/outline");
        ```

        Make sure the icon names you use in `bq-icon` or the framework wrappers match the icon set you provide.
      </Tab>

      <Tab title="Copy only the icons you need">
        If your app only uses a small subset of icons, you can manually copy just those SVG files into `src/assets` or another public folder and point `setBasePath()` there:

        ```ts
        import { setBasePath } from "@beeq/core";

        setBasePath("/svg");
        ```

        <Tip>
          Keep the original file names unchanged. BEEQ looks up icons by file name.
        </Tip>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Set up BEEQ in a standalone Angular app" titleSize="h3">
    For Angular `14+`, standalone components are the recommended setup for new work. Import the BEEQ components you need from `@beeq/angular/standalone`:

    <Tabs>
      <Tab title="Recommended: Standalone">
        Use direct component imports in standalone Angular apps:

        ```ts app.component.ts
        import { Component } from "@angular/core";
        import { BqButton, BqCard, BqInput } from "@beeq/angular/standalone";

        @Component({
          selector: "app-root",
          standalone: true,
          imports: [BqButton, BqCard, BqInput],
          templateUrl: "./app.component.html",
        })
        export class AppComponent {}
        ```
      </Tab>

      <Tab title="Existing NgModule app">
        If your app still uses NgModules, keep using `BeeQModule`:

        ```ts app.module.ts
        import { NgModule } from "@angular/core";
        import { BrowserModule } from "@angular/platform-browser";
        import { BeeQModule } from "@beeq/angular";

        import { AppComponent } from "./app.component";

        @NgModule({
          declarations: [AppComponent],
          imports: [BrowserModule, BeeQModule.forRoot()],
          bootstrap: [AppComponent],
        })
        export class AppModule {}
        ```
      </Tab>
    </Tabs>

    <Tip>
      If you are starting fresh, prefer standalone imports. Use `BeeQModule` only when you are integrating BEEQ into an existing NgModule-based application.
    </Tip>
  </Step>

  <Step title="Render your first component" titleSize="h3">
    Once styles and icons are configured, you can use BEEQ components directly in your Angular templates:

    ```html
    <bq-button (bqClick)="save()">
      Save changes
      <bq-icon name="floppy-disk" slot="suffix"></bq-icon>
    </bq-button>
    ```
  </Step>
</Steps>

That is enough to get started.

## Angular usage patterns with BEEQ

### Use Angular template bindings

BEEQ components render as standard custom elements in Angular templates, so inputs and outputs follow familiar Angular syntax:

```html
<bq-button [disabled]="isSaving" (bqClick)="save()">
  Save changes
</bq-button>
```

### Use `[(ngModel)]` with supported form controls

For template-driven forms in standalone Angular components, import `FormsModule` alongside the BEEQ controls you use, then add `ngDefaultControl` on the BEEQ control:

<CodeGroup>
  ```ts newsletter.component.ts highlight={2-3,9}
  import { Component } from "@angular/core";
  import { FormsModule } from "@angular/forms";
  import { BqInput } from "@beeq/angular/standalone";

  @Component({
    selector: "app-newsletter",
    standalone: true,
    imports: [FormsModule, BqInput],
    templateUrl: "./newsletter.component.html",
  })
  export class NewsletterComponent {
    name = "";
  }
  ```

  ```html newsletter.component.html highlight={3}
  <bq-input
    name="full-name"
    [(ngModel)]="name"
    ngDefaultControl
    placeholder="How should we call you?"
  >
    <label slot="label">Your name</label>
  </bq-input>
  ```
</CodeGroup>

### Use reactive forms when you need more control

Reactive forms work well with BEEQ controls when you want explicit form state and validation:

<CodeGroup>
  ```ts profile-form.component.ts
  import { Component } from "@angular/core";
  import { FormControl, FormGroup, ReactiveFormsModule } from "@angular/forms";
  import { BqInput } from "@beeq/angular/standalone";

  @Component({
    selector: "app-profile-form",
    standalone: true,
    imports: [ReactiveFormsModule, BqInput],
    templateUrl: "./profile-form.component.html",
  })
  export class ProfileFormComponent {
    profileForm = new FormGroup({
      givenName: new FormControl(""),
      familyName: new FormControl(""),
    });
  }
  ```

  ```html profile-form.component.html
  <form [formGroup]="profileForm">
    <bq-input formControlName="givenName" ngDefaultControl>
      <label slot="label">Given name</label>
    </bq-input>

    <bq-input formControlName="familyName" ngDefaultControl>
      <label slot="label">Family name</label>
    </bq-input>
  </form>
  ```
</CodeGroup>

### Keep NgModule setup for older Angular apps

If your Angular application is still module-based, BEEQ continues to support that flow through `BeeQModule`:

```ts app.module.ts
import { NgModule } from "@angular/core";
import { BrowserModule } from "@angular/platform-browser";
import { FormsModule } from "@angular/forms";
import { BeeQModule } from "@beeq/angular";

import { AppComponent } from "./app.component";

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule, FormsModule, BeeQModule.forRoot()],
  bootstrap: [AppComponent],
})
export class AppModule {}
```

## Complete example

This example combines icons, `ngModel`, and event handling in a standalone Angular app.

<CodeGroup>
  ```ts main.ts
  import { bootstrapApplication } from "@angular/platform-browser";

  import { AppComponent } from "./app/app.component";

  bootstrapApplication(AppComponent).catch((err) => console.error(err));
  ```

  ```ts app.component.ts
  import { Component } from "@angular/core";
  import { FormsModule } from "@angular/forms";
  import { BqButton, BqCard, BqIcon, BqInput } from "@beeq/angular/standalone";

  @Component({
    selector: "app-root",
    standalone: true,
    imports: [FormsModule, BqButton, BqCard, BqIcon, BqInput],
    templateUrl: "./app.component.html",
  })
  export class AppComponent {
    name = "";
    count = 0;

    increment() {
      this.count += 1;
    }
  }
  ```

  ```html app.component.html
  <bq-card>
    <h1>Hello, {{ name || "stranger" }}!</h1>

    <bq-input
      [(ngModel)]="name"
      ngDefaultControl
      placeholder="How should we call you?"
    >
      <bq-icon name="user" slot="prefix"></bq-icon>
    </bq-input>

    <bq-button (bqClick)="increment()">
      Give a thumbs up
      <bq-icon name="thumbs-up" slot="suffix"></bq-icon>
    </bq-button>

    <p>Total of thumbs: {{ count }}</p>
  </bq-card>
  ```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Styles are missing">
    Make sure `@beeq/core/dist/beeq/beeq.css` is loaded once globally, either from `angular.json` or from your main stylesheet.
  </Accordion>
  <Accordion title="Icons are not rendering">
    Check both of these:

    - the SVG files are being served by your Angular app
    - `setBasePath()` points to the same public URL where those files are available
  </Accordion>
  <Accordion title="Angular does not recognize the BEEQ components">
    In standalone apps, confirm that the specific BEEQ components are listed in the component `imports` array. In module-based apps, confirm that `BeeQModule.forRoot()` is imported.
  </Accordion>
  <Accordion title="`ngModel` or `formControlName` is not working">
    Make sure the relevant Angular forms module is imported and add `ngDefaultControl` on the BEEQ form control you bind.
  </Accordion>
  <Accordion title="I am using an older Angular setup and forms still fail">
    Some older Angular and Stencil output-target combinations have known compiler issues. If you hit a build-time issue around value accessors, check your Angular version first and then review your TypeScript and Angular compiler settings before disabling framework optimizations.
  </Accordion>
</AccordionGroup>
