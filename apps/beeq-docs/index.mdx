It is Endava’s design system and web component library for building consistent, accessible digital products.
It brings together framework-agnostic UI components, design foundations, and practical implementation guidance to help teams move faster and stay aligned.

## What is BEEQ?

BEEQ combines reusable UI components, design foundations, and practical guidance in one shared system. It helps designers and developers create consistent experiences, reduce duplication, and work with a common language across products and teams.

The component library is built with StencilJS and delivered as native HTML custom elements, making it portable across modern web environments. Styling and customization are designed to work cleanly with Tailwind CSS-based workflows.

## Why teams use BEEQ

A design system is a shared set of components, standards, and guidelines that helps teams design and build products more consistently.
It improves reuse, supports accessibility, and makes collaboration between design and development more effective.

<Tip>
  - **Consistency:** Create a shared UI language across products, teams, and brands.
  - **Efficiency:** Reuse proven building blocks instead of recreating patterns from scratch.
  - **Scalability:** Support everything from prototypes to enterprise-grade applications.
  - **Accessibility:** Build on foundations and components designed for inclusive product experiences.
</Tip>

## What makes BEEQ different

BEEQ is designed to be flexible, scalable, and practical across products, teams, and frontend stacks.

- **White-label and open-source**\
  BEEQ is designed to support different brands and products without imposing a fixed visual identity, while remaining transparent and easy to extend.
- **Framework-agnostic core**\
   BEEQ’s components are built as standard web components (using [StencilJS](https://stenciljs.com/)), meaning the same core can be used across _any_ major frontend framework—Angular, React, Vue—thanks to dedicated wrappers for each environment. Many popular libraries are often tied to only one framework (e.g., Material UI for React), while BEEQ is truly cross-framework.
- **Consistency at scale**\
   The system’s design guidelines and component library promote consistent UI/UX across large teams and multiple projects, no matter the front-end technology. Combined with its white-label nature, this enables organizations to roll out branded experiences at scale and speed.
- **Tailored for enterprise workflows**\
   The BEEQ monorepo includes tailored tooling ([NX](https://nx.dev/), [Volta](https://volta.sh/), [PNPM](https://pnpm.io/)) and integrated style presets like [Tailwind CSS](http://tailwindcss.com/), making modern development practices and streamlined CI/CD pipelines easy to adopt.
- **Developer experience**\
   Comprehensive docs, Storybook demos, and clear API design make BEEQ easy to adopt, extend, and maintain. The design system is as much about enabling _good process_ (including clear contribution flows and robust testing) as about the components themselves.
- **Transparent, community-driven development**\
   BEEQ welcomes contributions and is actively managed through open-source practices, with documented branching strategies and regular releases.
