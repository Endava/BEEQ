## What is BEEQ?

BEEQ combines reusable UI components, design foundations, and practical guidance in one shared system. It helps designers and developers create consistent experiences, reduce duplication, and work with a common language across products and teams.

The component library is built with StencilJS and delivered as native HTML custom elements, making it portable across modern web environments. Styling and customization are designed to work cleanly with TailwindCSS-based workflows.

## What is a design system?

A design system is a shared set of components, standards, and guidelines that helps teams design and build products more consistently. It improves reuse, supports accessibility, and makes collaboration between design and development more effective.

<Tip>
There are several benefits to using BEEQ as your design system:

 * **Consistency:** BEEQ makes sure everything looks the same, so your project has a nice, unified style.
 * **Efficiency:** It helps you work faster by giving you ready-to-use parts, so you don't have to start from scratch every time.
 * **Scalability:** Built with scalability in mind, BEEQ adapts seamlessly to projects of various sizes, from prototypes to enterprise-level applications.
 * **User-Centric:** BEEQ helps you make designs that not only work but also look good and make users happy.
</Tip>

## What makes BEEQ different?

BEEQ stands out from other UI component libraries in several key ways:

- **White-label and open-source**<br />
  BEEQ is specifically designed as a white-label system, making it highly customizable for different brands and products without imposing any particular style or branding. This flexibility is less common in libraries that enforce opinionated designs.

- **Framework-agnostic core**<br />
  BEEQ’s components are built as standard web components (using [StencilJS](https://stenciljs.com/)), meaning the same core can be used across *any* major frontend framework—Angular, React, Vue—thanks to dedicated wrappers for each environment. Many popular libraries are often tied to only one framework (e.g., Material UI for React), while BEEQ is truly cross-framework.


- **Accelerated consistency**<br />
  The system’s design guidelines and component library promote consistent UI/UX across large teams and multiple projects, no matter the frontend technology. Combined with its white-label nature, this enables organisations to roll out branded experiences at scale and speed.

- **Tailored for enterprise workflows**<br />
  The BEEQ monorepo includes tailored tooling ([NX](https://nx.dev/), [Volta](https://volta.sh/), [PNPM](https://pnpm.io/)) and integrated style presets like [Tailwind CSS](http://tailwindcss.com/), making modern development practices and streamlined CI/CD pipelines easy to adopt.

- **Extensive documentation and developer experience**<br />
  Comprehensive docs, Storybook demos, and clear API design make BEEQ easy to adopt, extend, and maintain. The design system is as much about enabling *good process* (including clear contribution flows and robust testing) as about the components themselves.

- **Transparent, community-driven development**<br />
  BEEQ welcomes contributions and is actively managed through open-source practices, with documented branching strategies and regular releases.
