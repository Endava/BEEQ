This page covers the three techniques you reach for when customizing a specific component: shadow DOM parts, global token overrides, and component-level CSS variables. For building a full custom theme or understanding the complete token system, see the [Theming](/theming) section.

<Note>
  The examples on this page use inline CSS for clarity. In your application, keep overrides in your own stylesheets or theme files.
</Note>

## Choose the right styling level

<CardGroup cols={3}>
  <Card title="Global design tokens" icon="palette">
    Override shared tokens such as `--bq-brand` when you want to change the look of your whole product.
  </Card>
  <Card title="Component CSS variables" icon="sliders">
    Override variables such as `--bq-button--large-paddingY` when you want to change a specific component pattern.
  </Card>
  <Card title="Shadow DOM parts" icon="layer-group">
    Target exposed parts with `::part()` when you need to style internal component sections such as labels or wrappers.
  </Card>
</CardGroup>

## Component shadow DOM parts

Shadow DOM encapsulates a web component's internal structure so it is less affected by the surrounding page styles and scripts. One of the key styling hooks it exposes is the `part` attribute.

A part is a named internal section of a component that can be styled from the outside. For example, `bq-button` exposes internal parts such as the button wrapper and label:

```html
<!-- bq-button.tsx -->
<button part="button">
  <span class="bq-button__label" part="label">
    <slot />
  </span>
</button>
```

Once a part is exposed, you can style it with the `::part()` pseudo-element selector.

  In addition to styling parts, you can inspect them from JavaScript through the component's `shadowRoot` when you need to query internal elements programmatically.
</Tip>

```ts
const bqButton = document.querySelector(".custom-button");
const nativeButton = bqButton?.shadowRoot?.querySelector('[part="button"]');
```

## Global CSS custom properties

CSS custom properties, also known as CSS variables, let you define a value once and reuse it across your application. BEEQ relies on them heavily to provide a consistent visual system without requiring a CSS preprocessor.

When you want to change a shared brand decision across the whole product, override a global token such as `--bq-brand` at the root level:

```css
:root {
  --bq-brand: var(--bq-orange-600);
}
```

Many theme tokens for UI, stroke, text, and related states derive from `--bq-brand`, so a single change can cascade through the interface.

<CodeLivePreview code={`
  <style>
    bq-button::part(button) {
      --bq-ui--brand: var(--bq-orange-600) !important;
    }
  </style>

  <bq-button appearance="primary">
    Primary button
  </bq-button>
`}>
  <CodeGroup>
    ```css CSS icon=css

      --bq-brand: var(--bq-orange-600);
    }
    ```

    ```html HTML icon=html5
    <bq-button appearance="primary">
      Primary button
    </bq-button>
    ```

    ```tsx React icon=react

    import "./styles.css";

    export function Example() {
      return <BqButton appearance="primary">Primary button</BqButton>;
    ```

## Component-level CSS custom properties

Component-level CSS variables are scoped to a single component instance and do not affect the rest of the theme. For a full explanation of override strategies — including how to apply changes across all instances of a component at once — see [Component CSS Variables](/theming/component-css-variables).

In the example below, the second button overrides one brand token locally while the first keeps the theme default:

<CodeLivePreview code={`
  <style>
    .primary-magenta {
      --bq-ui--brand: var(--bq-magenta-600);
    }
  </style>

  <bq-button appearance="primary">
    Primary button
  </bq-button>

  <bq-button appearance="primary" class="primary-magenta">
    Primary custom
  </bq-button>
`}>
  <CodeGroup>
    ```css CSS
    .primary-magenta {
      --bq-ui--brand: var(--bq-magenta-600);
    }
    ```

    ```html HTML icon=html5
    <bq-button appearance="primary">
      Primary button
    </bq-button>

    <bq-button appearance="primary" class="primary-magenta">
      Primary custom
    </bq-button>
    ```

    ```tsx React icon=react

        <>
          <BqButton appearance="primary">Primary button</BqButton>
    ```


Some components also expose CSS variables that only apply to that component. These are not global design tokens. They are local styling hooks documented in each component page's API reference.

## Where to find component styling hooks

Not every component exposes the same CSS variables or parts. When a component does expose them, you can find them in:

- the component page's API reference
- the component source in [GitHub](https://github.com/Endava/BEEQ/tree/main/packages/beeq/src/components) under `packages/beeq/src/components/**/scss/*.variables.scss`

## Best practices

<CardGroup cols={2}>
  <Card title="Start with tokens" icon="check" iconType="solid">
    Use global design tokens first when the change should apply across the product. This keeps the theme consistent and easier to maintain.
  </Card>
  <Card title="Scope local exceptions" icon="check" iconType="solid">
    Use classes or component selectors for one-off overrides so you do not accidentally restyle every instance.
  </Card>
  <Card title="Use parts sparingly" icon="check" iconType="solid">
    Reach for `::part()` when a component-level variable is not enough. It is powerful, but it couples your styles more closely to component internals.
  </Card>
  <Card title="Test both modes" icon="check" iconType="solid">
    Review your overrides in both light and dark mode to make sure contrast, hierarchy, and focus states still work well.
  </Card>
</CardGroup>
